Suggested working order

Open the reference prototype side-by-side with your editor.

Keep reference-prototype/index.html open in a browser tab as you work — every rebuilt screen should match it.

Install the design-system foundation.

Copy globals.css, tailwind.config.ts, fonts.ts, and the components/ui/* files into src/. This replaces all Aurora Glass visual primitives.

Install the shell.

Drop in features/layout/*. Wrap app/layout.tsx with <RouteProvider> and render <AppShell />.

Migrate screens in the order listed in HANDOFF.md.

Dashboard → Agents → Workflows → Marketplace. Each screen spec in design-system/screens.md tells you exactly which components to compose.

Replace mock data with real API calls.

Mock shapes in types/index.ts match the existing backend; grep for // TODO: real data in each feature.
